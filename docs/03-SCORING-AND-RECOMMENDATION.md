# 03. 광고주 Scoring Logic & 광고상품 Recommendation Logic

> Phase 1 산출물 / 모든 항목·가중치·계산식은 `ScoringConfig`로 관리(코드 하드코딩 금지, ADR-007)

## 1. 광고주 적합도 스코어링 (총 100점)

### 1.1 항목·가중치 (기본 config v1)
| 코드 | 평가항목 | 최대점 | 세부지표 |
|---|---|---|---|
| MKT | 시장성 | 15 | 시장규모·성장성·계절성·광고수요·트렌드 |
| TGT | 고객 타겟 적합도 | 20 | 회원연령·자녀연령·가족고객·지역·관심카테고리 적합도 |
| BHV | 회원 행동 데이터 적합도 | 20 | 조회·찜·장바구니·구매·예약·검색량·재구매율 |
| PERF | 광고 성과 예상도 | 15 | 예상CTR·CVR·유사광고주성과·카테고리평균·상품적합도 |
| ACT | 광고주 활동성 | 10 | 최근광고·프로모션빈도·신제품·확장·디지털마케팅 |
| BDG | 예산 적합도 | 10 | 예상예산·상품가격적합·캠페인규모·집행가능성 |
| CMP | 경쟁 기회 | 10 | 경쟁광고활동·경쟁강도·차별화가능성·회원접점 |
| | **합계** | **100** | |

### 1.2 계산 모델
각 항목 점수 = `Σ(지표 정규화값[0..1] × 지표 가중치) × 항목 최대점`.
- 지표별 `calc_method`(config): `linear`, `threshold`, `lookup(벤치마크)`, `agg_ratio(집계비율)`.
- 내부 데이터 기반 지표(BHV 등)는 **집계 값**만 사용(비식별). 외부 판단 지표(MKT/CMP)는 Agent 산출 + 신뢰도.

```
총점 = MKT + TGT + BHV + PERF + ACT + BDG + CMP   (0..100)
```

### 1.3 등급 매핑 (config)
| 총점 | 등급 | 액션 |
|---|---|---|
| ≥90 | S | 즉시 영업 추천 |
| 80–89 | A | 우선 영업 추천 |
| 70–79 | B | 조건부 추천 |
| 60–69 | C | 추가 검토 |
| <60 | D | 추천 보류 |

### 1.4 설명가능성 산출물 (항목별 저장)
각 광고주 점수 계산 시 `AdvertiserScore(총점·등급·scoring_version)` + `ScoreFactor[]`(항목코드·획득점·최대점·정규화 근거·사용 데이터 참조·사실/추론 플래그·신뢰도)를 저장 → UI Drawer에서 그대로 노출.

### 1.5 재현성 & 재계산
- 점수는 항상 특정 `ScoringVersion`으로 계산·태깅.
- 가중치 변경 → 새 version 발행 → 백그라운드 재계산(Celery). 과거 결과는 이전 version으로 보존.

## 2. 광고상품 추천 로직 (4상품 각 100점)

### 2.1 대상 상품
메인배너 · 서브배너 · 카테고리 광고 · 스플래쉬 광고 (정의/특징/적합목적/추천광고주/추천조건은 스펙 원문 기준, DB `AdProduct`에 관리)

### 2.2 항목·가중치 (기본 config v1)
| 코드 | 평가항목 | 최대점 |
|---|---|---|
| GOAL | 캠페인 목적 적합도 | 20 |
| CAT | 카테고리 적합도 | 20 |
| AUD | 타겟 고객 적합도 | 20 |
| BDG | 예산 적합도 | 15 |
| EXPO | 노출 방식 적합도 | 10 |
| TIME | 기간·긴급성 | 10 |
| HIST | 과거 성과 적합도 | 5 |
| | **합계** | **100** |

### 2.3 상품별 가점 규칙(Rule Boost)
`AdProductCategoryRule`로 관리. 조건 매칭 시 특정 항목 점수를 상향(스펙의 우선 추천 규칙 반영).
- **메인배너↑**: 인지도 목적·전국타겟·대형브랜드·대규모프로모션·고예산·신상품.
- **서브배너↑**: 중간예산·지속노출·단일상품/프로모션·효율중시·중견/중소.
- **카테고리광고↑**: 업종-카테고리 일치·전환목적·타겟팅·지역/상품홍보·효율중시.
- **스플래쉬↑**: 단기집중·이벤트·신규론칭·강한비주얼·긴급·명확한 시작/종료일·공공/축제.

### 2.4 조합 추천 (Combination)
단일 1·2순위 + **패키지 추천**:
| 패키지 | 구성 | 적합 목적 |
|---|---|---|
| 브랜드 인지도 | 메인 + 스플래쉬 | 신규 론칭·대형 프로모션·초기 인지 |
| 전환 중심 | 카테고리 + 서브 | 예약/구매 전환·타겟팅 |
| 대형 캠페인 | 메인+서브+카테고리+스플래쉬 | 전국·시즌·축제·대형 브랜드 |
| 효율형 | 카테고리 + 서브 | 제한 예산·명확 타겟·CTR/CVR 중심 |

조합 추천 시 **각 상품의 역할**을 명시(예: 스플래쉬=초기 인지, 카테고리=예약 전환).

### 2.5 예상 성과 산출
`BenchmarkTable`(카테고리×상품별 평균 CTR/CVR/CPM/전환단가) + 유사 광고주 성과 가중평균으로 추정:
```
예상 노출 = f(상품 인벤토리, 예산, CPM)
예상 클릭 = 예상 노출 × 예상 CTR
예상 전환 = 예상 클릭 × 예상 CVR
예상 매출 = 예상 전환 × 카테고리 평균 객단가
예상 ROI  = (예상 매출 − 광고비) / 광고비 × 100
```
모든 예상치는 **"실데이터 기반 / 가정"** 라벨과 신뢰도(%)를 함께 표기.

### 2.6 최종 추천 Output 스키마(핵심 필드)
광고주명·카테고리·광고목적·종합점수·등급·1순위상품(+점수·이유)·2순위·추천조합(+역할)·상품별 적합도·예상(노출/클릭/CTR/전환/CVR/매출/ROI)·추천 집행기간·시기·타겟·소재방향·리스크·데이터신뢰도.
